## 网络请求检查

### 打开浏览器开发者工具 -> Network

**请求信息**：
- URL: `http://localhost:5555/api/article`
- Method: `POST`
- Headers: 应包含 `Authorization` token
- Request Body: 
```json
{
  "title": "文章标题",
  "sub_title": "副标题",
  "type": "分类名称",
  "cover": "封面URL",
  "content": "文章内容",
  "evidently": 1,
  "status": 1
}
```

**响应信息**：
- Status: `200` 或 `201`
- Response Body: 包含创建的文章数据

## 常见问题排查

### 问题1: 分类下拉框不显示文字
**原因**: afterFetch未正确转换数据
**解决**: 已修复，使用afterFetch转换为{label, value}格式

### 问题2: 提交没有反应
**可能原因**:
1. 表单验证失败（查看控制台和字段错误提示）
2. 后端服务未启动
3. 网络请求失败（查看Network面板）
4. 认证token过期（查看Network面板返回401）

**排查步骤**:
1. 打开浏览器控制台查看日志
2. 查看Network面板的请求状态
3. 确认后端服务运行正常
4. 确认已登录且token有效

### 问题3: 提交成功但表单未重置
**原因**: formApi.resetForm()可能异步失败
**解决**: 已添加await确保重置完成
